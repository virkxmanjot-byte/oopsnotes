Introduction to OOP – Classes and Objects
OOP BasicsClassObject
▼

Key Definitions

OOP (Object-Oriented Programming): A programming paradigm that organizes software design around data (objects) rather than functions and logic.
Class: A user-defined blueprint/template from which objects are created. No memory is allocated when a class is defined.
Object: An instance of a class. Memory is allocated when an object is created. An object has State, Behavior, and Identity.

Core OOP Concepts (Pillars)

EncapsulationWrapping data + functions in one unit (class).
AbstractionHiding complex details; showing only essential features.
InheritanceChild class acquires properties of parent class.
PolymorphismOne name, many forms. Same function behaves differently.

3 Properties of Every Object

  • State – Represented by attributes (data members). E.g., colour, name, balance.
  • Behavior – Represented by methods (member functions). E.g., deposit(), display().
  • Identity – Unique name that distinguishes the object.

Why OOP?

  • Faster development and easier debugging
  • Promotes code reuse (DRY – Don't Repeat Yourself)
  • Clear structure using classes and objects
  • Easier to maintain and modify large programs

Basic Class Syntax

class ClassName {
  // Data members (variables)
  // Member functions (methods)
public:
  dataType variable;
  void functionName() { }
};

One-Line Revision Notes

OOP = Data-centric programming Class = Blueprint Object = Instance of class 4 Pillars: Encapsulation, Abstraction, Inheritance, Polymorphism

Summary

OOP organizes code around objects which have state and behavior. A class is a blueprint; creating an object allocates memory. The 4 pillars of OOP form the foundation of all C++ object-oriented design.

Methods in OOP

Methods are functions that belong to a class. They can be defined inside or outside the class definition. Accessed using the dot (.) operator on an object.

Message Passing

Message Passing: When one object calls a method on another object to request it to perform an action. This is the mechanism of object communication in OOP.

Example: f1.turnOn(); — object f1 receives a message to execute turnOn().

Abstraction

Abstraction: Showing only essential features while hiding internal complexity. Users interact with what is important, not how it works internally.
  • You drive a car without knowing the engine internals
  • You use a smartphone without knowing circuit boards
  • In C++: achieved using classes, abstract classes, and interfaces
  • Operates at design level

Encapsulation

Encapsulation: Wrapping data members and member functions into a single unit (class). Prevents direct access to data. Achieved using private data + public getter/setter methods.
  • Acts as a protective shield around data
  • Operates at implementation level
  • Example: Bank account balance should only change via deposit()/withdraw()

Abstraction vs. Encapsulation

Code Example

class Student {
public:
    string name;
    int age;

        cout << "Name: " << name << endl;

    }

Inheritance

Inheritance: An OOP concept where a child class (subclass/derived) acquires properties and behaviors of a parent class (superclass/base class).

Why use Inheritance?

  • Code Reusability – Avoid rewriting the same code
  • Extensibility – Add features without modifying existing classes
  • Organized Code – Hierarchical relationships
  • Polymorphism Support – Enables function overriding

Basic Inheritance Syntax

class Base {
    // Data and functions
};
class Derived : access_specifier Base {
    // Additional data and functions
};

Types of Inheritance

Hierarchical1 Base → Many DerivedShape → Circle, Square
HybridMix of multiple + multilevelPerson → Student, Employee → WorkingStudent

Polymorphism

Polymorphism: Derived from Greek – "poly" (many) + "morph" (forms). The same function name performs different actions based on context/object.
TypeHow AchievedDecision Time
Compile-Time (Static)Function overloading, Operator overloadingCompile time

Abstract Class

Abstract Class: A class that cannot be instantiated directly. It contains at least one pure virtual function (= 0).
class Animal {
public:
    virtual void sound() = 0;  // Pure virtual function
};
class Dog : public Animal {
public:
    void sound() { cout << "Bark!"; }
};
int main() {
    Dog d;
    d.sound();  // Output: Bark!
}

Data Members

Data Members: Variables declared inside a class that represent the state/properties of an object. They hold information about the object.

Types:

  • Instance (Non-static): Each object gets its own copy. Belongs to the object.
  • Static: Shared by all objects. Only one copy in memory. Accessed as ClassName::variable.

Member Functions

Member Functions: Functions inside a class that define behavior of the object. Think: Data members = What object HAS; Member functions = What object DOES.

Default Arguments

Default Arguments: Values automatically used when the caller does not provide them. Must be rightmost parameters.
void f(int a = 10) {
    cout << a << endl;
}
int main() {
    f();     // Output: 10
    f(221);  // Output: 221
}

Static Member Example

class Student {
public:

};

    Student s1, s2, s3;
    cout << "Total: " << Student::totalStudents; // Output: 3

Function Overloading & Friend Functions
Function OverloadingFriend FunctionCompile-time Polymorphism
▼

Function Overloading

Function Overloading: Defining multiple functions with the same name but different parameter lists (different number, type, or order of parameters). It is a form of compile-time polymorphism.

Rules:

  • Functions must differ in type, number, or order of parameters
  • Return type alone cannot differentiate overloaded functions
  • Compiler selects the right function at compile time

Function Overloading Example

#include <iostream>
using namespace std;
class Addition {
public:
    int add(int a, int b) { return a + b; }
    float add(float a, float b) { return a + b; }
    int add(int a, int b, int c) { return a + b + c; }
};
int main() {
    Addition obj;
    cout << obj.add(5, 3) << endl;        // int: 8
    cout << obj.add(2.5f, 3.5f) << endl;  // float: 6.0
    cout << obj.add(1, 2, 3) << endl;     // int: 6
}

Friend Functions

Friend Function: A function that is NOT a member of the class but is granted access to private and protected members of the class. Declared using the keyword friend inside the class.

Key Points:

  • Declared inside the class with friend keyword
  • Defined outside the class WITHOUT class name or scope resolution (::)
  • Can access private members directly
  • NOT a member function – called without an object
  • Not inherited

Friend Function Example

class Box {
    int width;
public:
    Box(int w) : width(w) {}
    friend void printWidth(Box b);  // Friend declaration
};
void printWidth(Box b) {  // Friend definition (no class name)
    cout << "Width: " << b.width;  // Can access private member
}
int main() {
    Box b(10);
    printWidth(b);  // Output: Width: 10
}

One-Line Revision Notes

Overloading = same name, diff params Friend = non-member with private access Friend not inherited Overloading is compile-time polymorphism

Static Members of a Class
Static MembersStatic Functions
▼

Static Data Members

Static Data Member: A class-level variable shared by all objects. Only one copy exists in memory regardless of how many objects are created.
  • Declared with static keyword inside class
  • Must be defined and initialized outside the class
  • Accessed as ClassName::staticVar
  • Common use: counting objects, shared configuration

Static Member Functions

Static Member Function: A function that belongs to the class (not to any object). Can only access static data members and other static functions. Called using ClassName::function().
  • Cannot use this pointer
  • Cannot access non-static data members
  • Useful for utility functions that don't need object state

Example

class Counter {
    static int count;
public:
    Counter() { count++; }
    static int getCount() { return count; }  // Static function
};
int Counter::count = 0;  // Definition outside class

int main() {
    Counter c1, c2, c3;
    cout << Counter::getCount();  // Output: 3
}

One-Line Revision Notes

static = class-level, not object-level Static fn can't use 'this' Static defined outside: int ClassName::var = val Call: ClassName::staticFn()

Type Conversion in C++
Type ConversionCastingstatic_castdynamic_cast
▼

Types of Conversion

TypeDescriptionExample
Implicit (Automatic)Compiler converts automaticallyint → double
Explicit (Casting)Programmer forces conversion(int)x or static_cast<int>(x)

Widening Conversion: Safe – smaller to larger type (int → double)

Narrowing Conversion: Risky – larger to smaller type (double → int) → data loss!

C++ Cast Operators

static_castMost common safe cast. Numeric conversions, upcasting. Checked at compile time.
dynamic_castRuntime safe cast. Used in polymorphism. Needs virtual functions. Returns nullptr if fails.
const_castAdds or removes const qualifier. Modifying originally const causes undefined behavior.
reinterpret_castLow-level bit-level cast. Converts unrelated pointer types. Dangerous – use only in systems programming.

static_cast Example

double x = 8.9;
int y = static_cast<int>(x);  // y = 8 (truncated)
cout << y;  // Output: 8

// Fix integer division:
int a = 5, b = 2;
double result = static_cast<double>(a) / b;  // 2.5

dynamic_cast Example

class Base { public: virtual void show() {} };
class Derived : public Base { public: void display() { cout << "Derived"; } };

Base* b = new Derived();
Derived* d = dynamic_cast<Derived*>(b);
if (d != nullptr) {
    d->display();  // Output: Derived
}

User-Defined Conversion

  • Conversion Constructor: Single-parameter constructor allows implicit type conversion
  • Conversion Operator: operator int() converts class object to int
  • explicit keyword: Prevents unwanted implicit conversions

One-Line Revision Notes

double→int = narrowing = data loss static_cast = compile-time safe cast dynamic_cast = runtime safe (needs virtual) reinterpret_cast = dangerous low-level

Constant & Volatile Objects
constvolatileConstant Objects
▼

Constant Objects

Constant Object: An object declared with the const keyword. Its data members cannot be modified after creation.
  • Declared as: const ClassName obj;
  • Can only call const member functions
  • Provides data safety and program reliability
  • Useful for bank account details, configuration values

Constant Member Functions

Const Member Function: A function that promises not to modify any data member of the class. Declared with const after the parameter list.
class Test {
public:
    void show() const {  // Const member function
        cout << "This is a constant function";
    }
};
int main() {
    const Test t1;
    t1.show();  // Allowed: const obj calling const function
}

Rules: Const Object + Const Function

  • A const object can ONLY call const member functions
  • A const function cannot modify data members
  • A regular object can call both const and non-const functions

Volatile Objects

volatile: Tells the compiler the variable can change at any time from outside the program (hardware, OS). Prevents compiler from optimizing/caching that variable.
  • Used in embedded systems and hardware programming
  • Syntax: volatile int x;

One-Line Revision Notes

const object → only const functions const fn can't modify data volatile = value may change externally const fn syntax: void fn() const { }

Nested Classes & Local Classes
Nested ClassLocal Class
▼

Nested Class

Nested Class: A class defined inside another class. The inner class is a member of the outer class.
  • Inner class can access static members and types of outer class
  • Inner class object created separately: Outer::Inner obj;
  • Used to logically group related classes (e.g., Node inside LinkedList)
class Outer {
public:
    class Inner {        // Nested class
    public:
        void show() { cout << "Inner class"; }
    };
};
int main() {
    Outer::Inner obj;   // Creating inner class object
    obj.show();
}

Local Class

Local Class: A class defined inside a function. Its scope is limited to that function only.
  • Cannot use non-static local variables of the enclosing function
  • Can use static variables of the enclosing function
  • All member functions must be defined inside the local class
  • Cannot have static data members
void myFunction() {
    class Local {    // Local class
    public:
        void display() { cout << "Local class"; }
    };
    Local obj;
    obj.display();
}

Key Differences

FeatureNested ClassLocal Class
Defined insideAnother classA function
ScopeOuter class scopeFunction scope only
AccessOuter::InnerOnly inside function

Parameterized Constructor & Dynamic Memory Allocation
Parameterized Constructornew operatorDynamic Memory
▼

Parameterized Constructor

Parameterized Constructor: A constructor that accepts arguments/parameters. Used to initialize objects with different values at the time of object creation.
class Student {
    int marks;
public:
    Student(int m) {   // Parameterized constructor
        marks = m;
    }
    void display() { cout << "Marks: " << marks; }
};
int main() {
    Student s1(90);  // Object created with value 90
    s1.display();    // Output: Marks: 90
}

Constructor with Dynamic Memory Allocation

When the amount of memory required is not known at compile time, a constructor can allocate memory dynamically using the new operator.
class Array {
    int *ptr;
    int size;
public:
    Array(int s) {
        size = s;
        ptr = new int[size];  // Dynamic allocation in constructor
        cout << "Memory allocated for " << size << " elements";
    }
    ~Array() {
        delete[] ptr;  // Always free in destructor
    }
};
⚠️ Always pair new with delete (in destructor) to prevent memory leaks!

One-Line Revision Notes

Parameterized constructor accepts args new = heap allocation; delete = free new[] for arrays; delete[] for array cleanup Dynamic allocation done when size unknown at compile time

Copy Constructor

Copy Constructor: A special constructor that creates a new object as a copy of an existing object. Takes a reference to an object of the same class as parameter.

Syntax: ClassName(const ClassName &obj) { }

When is Copy Constructor Called?

  • When an object is initialized from another: Student s2 = s1;
  • When an object is passed by value to a function
  • When a function returns an object by value

Shallow Copy vs Deep Copy

AspectShallow CopyDeep Copy
HowCompiler-provided defaultUser-defined copy constructor
PointersBoth objects point to SAME memoryEach object has its own memory
RiskDangling pointer problemSafe – independent copies
When neededNo dynamic memoryClass has pointer/dynamic members

Copy Constructor Example

#include <iostream>
using namespace std;
class Student {
    string name;
    int marks;
public:
    Student(string n, int m) : name(n), marks(m) {}
    // Copy constructor
    Student(const Student &s) {
        name = s.name;
        marks = s.marks;
        cout << "Copy constructor called";
    }
    void display() { cout << name << " " << marks; }
};
int main() {
    Student s1("Rahul", 90);
    Student s2 = s1;   // Copy constructor called
    s2.display();
}

One-Line Revision Notes

Copy constructor syntax: ClassName(const ClassName &obj) Shallow copy = same pointer (dangerous) Deep copy = new independent memory Must define if class has dynamic/pointer members

Destructor
DestructorMemory Management
▼

Destructor

Destructor: A special member function automatically called when an object goes out of scope or is deleted. Used to release resources (memory, file handles, etc.).

Syntax: ~ClassName() { }

Key Properties

  • Same name as class with a tilde (~) prefix
  • No return type, no parameters
  • Cannot be overloaded (only one destructor)
  • Called automatically when object goes out of scope
  • Called in reverse order of construction

Destructor Example

class Array {
    int *ptr;
public:
    Array(int size) {
        ptr = new int[size];
        cout << "Constructor: Memory allocated";
    }
    ~Array() {         // Destructor
        delete[] ptr;  // Free dynamic memory
        cout << "Destructor: Memory freed";
    }
};
int main() {
    Array a(5);
}  // Destructor called automatically here
⚠️ If you use new in the constructor, always use delete in the destructor to prevent memory leaks!

Virtual Destructor

When using polymorphism (base class pointer to derived object), declare the destructor as virtual to ensure the correct derived class destructor is called.

One-Line Revision Notes

Destructor = ~ClassName() Auto-called when object goes out of scope Cannot be overloaded Use virtual destructor in polymorphism

Operator Overloading

Operator Overloading

Operator Overloading: A form of compile-time polymorphism that allows existing C++ operators (+, -, *, <<, >> etc.) to be used with user-defined types (class objects).

Syntax:

return_type operator symbol (parameter_list) {
    // body
}

Operators That CANNOT Be Overloaded

❌ Cannot overload: :: (scope resolution), . (member access), .* (pointer to member), ?: (ternary), sizeof

Example: Overloading + Operator

class Complex {
    float real, imag;
public:
    Complex(float r = 0, float i = 0) : real(r), imag(i) {}
    Complex operator+(Complex c) {   // Overload +
        return Complex(real + c.real, imag + c.imag);
    }
    void display() { cout << real << "+" << imag << "i"; }
};
int main() {
    Complex c1(3, 4), c2(1, 2);
    Complex c3 = c1 + c2;  // Calls operator+
    c3.display();  // Output: 4+6i
}

Unary Operator Overloading (++)

class Counter {
    int count;
public:
    Counter(int c) : count(c) {}
    Counter operator++() {   // Pre-increment
        ++count;
        return *this;
    }
    void display() { cout << count; }
};
int main() {
    Counter c(5);
    ++c;
    c.display();  // Output: 6
}

One-Line Revision Notes

Operator overloading = compile-time polymorphism . :: .* sizeof ?: cannot be overloaded Syntax: return_type operator+(param) { } Can be member function or friend function

Why Use Friend for Operator Overloading?

When using a member function for operator overloading, the left operand must be an object of the class. A friend function removes this restriction and allows both operands to be of different types or in any order.

Friend Function Operator Overloading Example

class Complex {
    float real, imag;
public:
    Complex(float r = 0, float i = 0) : real(r), imag(i) {}
    friend Complex operator+(Complex c1, Complex c2);  // Friend declaration
    void display() { cout << real << "+" << imag << "i"; }
};
Complex operator+(Complex c1, Complex c2) {  // Friend definition
    return Complex(c1.real + c2.real, c1.imag + c2.imag);
}
int main() {
    Complex c1(2, 3), c2(4, 5);
    Complex c3 = c1 + c2;
    c3.display();  // Output: 6+8i
}

Member vs Friend for Operator Overloading

AspectMember FunctionFriend Function
Left operandMust be class objectCan be any type
Parametersn-1 params for n-ary opn params for n-ary op
AccessHas direct accessAccesses via friend
Symmetric ops (+, ==)Asymmetric behaviorRecommended (symmetric)

One-Line Revision Notes

Friend op takes 2 params for binary op Friend preferred for symmetric operators (+, -, ==) << and >> MUST be overloaded via friend

Type Conversion (Advanced) – Implicit & Explicit
Type ConversionWideningNarrowing
▼

Implicit vs Explicit Conversion

Implicit Conversion: Compiler automatically converts one type to another. Safe for widening conversions (int→double) but risky for narrowing (double→int).
Explicit Conversion (Casting): Programmer forces conversion using cast operators. Gives full control and improves readability.

Common Pitfall

int totalMarks = 85;
int subjects = 2;
double average = totalMarks / subjects;  // ❌ Wrong! = 42.0 (int division)
// Fix:
double average = static_cast<double>(totalMarks) / subjects;  // ✅ = 42.5

Conversion Constructor

class Number {
    int value;
public:
    Number(int x) { value = x; }  // Conversion constructor
    void show() { cout << value; }
};
int main() {
    Number obj = 10;  // Implicit: int → Number via constructor
    obj.show();       // Output: 10
}

Conversion Operator

class Number {
    int value;
public:
    Number(int x) : value(x) {}
    operator int() { return value; }  // Conversion operator
};
int main() {
    Number obj(50);
    int x = obj;  // Number → int via conversion operator
    cout << x;    // Output: 50
}

One-Line Revision Notes

int/int = integer division (no decimal) Use static_cast to fix integer division Conversion constructor: single param constructor Conversion operator: operator type() { return val; }

explicitNamespacestd

Explicit Constructor

Explicit Constructor: A constructor declared with the explicit keyword that prevents implicit type conversion. Forces the programmer to use direct initialization.

Why Use explicit?

  • Prevents unintended object creation from wrong types
  • Improves type safety and program reliability
  • Recommended for single-parameter constructors
  • Used in safety-critical systems and large software projects

Example

// Without explicit - allows dangerous implicit conversion:
class Test {
public:
    Test(int x) {}  // Implicit conversion possible
};
Test t = 10;  // ✅ Works (implicit: int → Test)

// With explicit - safe:
class Safe {
public:
    explicit Safe(int x) {}  // Prevents implicit conversion
};
Safe s(10);   // ✅ Valid (direct initialization)
Safe s = 10;  // ❌ Error! Not allowed with explicit

Namespaces

Namespace: A named region that groups related identifiers to avoid naming conflicts. C++ Standard Library uses the std namespace.
#include <iostream>
int main() {
    std::cout << "Hello";  // Using scope resolution
    return 0;
}
// OR:
using namespace std;  // Import entire std namespace
cout << "Hello";  // Now no need for std::
⚠️ Avoid using namespace std; in large programs to prevent naming conflicts. Use std:: prefix instead.

Inheritance in C++ – All Types

Syntax of Inheritance

class DerivedClass : access_specifier BaseClass {

Single Inheritance

One base class → one derived class. Simplest form.

class Animal { public: void eat() { cout << "Eating"; } };
class Dog : public Animal { public: void bark() { cout << "Woof"; } };
int main() {
    Dog d;
    d.eat();   // Inherited
    d.bark();  // Own
}

Multiple Inheritance

One derived class inherits from multiple base classes. Bases separated by comma.

class A { public: void showA() { cout << "A"; } };
class B { public: void showB() { cout << "B"; } };
class C : public A, public B { };
int main() {
    C obj;
    obj.showA();  // From A
    obj.showB();  // From B
}

Multilevel Inheritance

A → B → C (chain). Derived class becomes base for another class.

class Vehicle { };
class Car : public Vehicle { };
class SportsCar : public Car { };  // Multilevel

Hierarchical Inheritance

One base → multiple derived classes.

class Shape { };
class Circle : public Shape { };
class Rectangle : public Shape { };
class Triangle : public Shape { };

Hybrid Inheritance

Combination of two or more types of inheritance. Often combines hierarchical + multiple.

⚠️ Hybrid inheritance can lead to the Diamond Problem. Use virtual inheritance to solve it.

Public Inheritance ("IS-A" Relationship)

Public members of base remain public in derived. Protected remain protected. Represents an IS-A relationship (e.g., Dog IS-A Animal).
class Base { public: int x; void display() { cout << x; } };
class Derived : public Base {
public:
    void setValue(int val) { x = val; }  // Can access base's public x
};
int main() {
    Derived obj;
    obj.setValue(10);
    obj.display();   // ✅ Works: public stays public
    // obj.x = 20;  // ✅ Also works
}

Private Inheritance ("HAS-A" / "Implemented-in-terms-of")

ALL public and protected members of base become PRIVATE in derived. Cannot be accessed from outside the derived class. Represents a HAS-A relationship.
class Derived : private Base {
public:
    void setValue(int val) { x = val; }  // ✅ Accessible inside derived
    void show() { display(); }           // ✅ Accessible inside derived
};
int main() {
    Derived obj;
    obj.setValue(20);
    obj.show();
    // obj.display(); // ❌ Error! Private inheritance hides it
}

Virtual Functions & Runtime Polymorphism
Virtual FunctionRuntime PolymorphismLate Binding
▼

Virtual Function

Virtual Function: A member function declared with the virtual keyword in a base class. When accessed through a base class pointer pointing to a derived class object, the derived class version is called (late binding).

Why Virtual Functions?

  • Enables Runtime Polymorphism (late binding / dynamic dispatch)
  • Allows base class pointers to call derived class methods
  • Forms the basis of OOP design patterns

Virtual Function Example

#include <iostream>
using namespace std;
class Base {
public:
    virtual void display() {  // Virtual function
        cout << "Base class function";
    }
};
class Derived : public Base {
public:
    void display() override {  // Override base class function
        cout << "Derived class function";
    }
};
int main() {
    Base* basePtr;
    Derived derivedObj;
    basePtr = &derivedObj;
    basePtr->display();  // Calls Derived's display (runtime decision)
    // Output: Derived class function
}

Without Virtual (Early Binding)

// Without virtual keyword:
Base* b = new Derived();
b->display();  // Calls BASE display (wrong!)

// With virtual keyword:
b->display();  // Calls DERIVED display (correct!)

VTable (Virtual Table)

The compiler maintains a VTable (virtual function pointer table) for each class with virtual functions. When a virtual function is called via a pointer, the program looks up the VTable at runtime to find the correct function.

override keyword

override keyword tells the compiler to check that the function actually overrides a base class virtual function. Prevents typos from silently breaking polymorphism.

One-Line Revision Notes

virtual = runtime binding (late binding) Use override to confirm overriding VTable used for dynamic dispatch Base* ptr = new Derived() → virtual needed for correct behavior

Virtual Destructor

When deleting a derived class object through a base class pointer, declare the destructor as virtual to ensure the derived class destructor is called properly.
class Base {
public:
    virtual ~Base() { cout << "Base destructor"; }  // Virtual destructor
};
class Derived : public Base {
public:
    ~Derived() { cout << "Derived destructor"; }
};
int main() {
    Base* b = new Derived();
    delete b;  // Without virtual: only Base destructor called (MEMORY LEAK!)
               // With virtual: Derived destructor → Base destructor (CORRECT)
}
⚠️ Always declare destructor as virtual when using polymorphism to prevent memory leaks!

Pure Virtual Function: A virtual function with no implementation in the base class, declared with = 0. Forces every derived class to provide its own implementation.

Exception Handling: A mechanism to handle runtime errors gracefully without crashing the program. Uses three keywords: try, throw, and catch.

    catch (const char* msg) {

    }
    catch (...) {   // Catch-all block

    }
    return 0;
}

Exception Specification

Exception Specification: A contract that tells whether a function may throw an exception. In modern C++, this is done using noexcept.

Old vs Modern Style

noexcept — A promise that the function will NOT throw any exception. Allows compiler optimizations and improves performance.

    cout << "This function never throws";
}

std::terminate()

std::terminate(): A C++ function called when exception handling cannot continue safely. Immediately stops the program.

When is std::terminate() called?

  • A noexcept function throws an exception (violation)
  • An exception is thrown during stack unwinding (destructor throws)
  • An exception is not caught by any handler

noexcept Violation Example

    throw 10;   // ❌ Violation! noexcept function cannot throw
}
int main() {
    show();  // Calls std::terminate() → program terminates!
}
⚠️ If a destructor throws during stack unwinding (another exception already active), std::terminate() is called — two active exceptions = program death!

Templates, Down Casting & Cross Casting

Upcasting (Derived → Base)

Upcasting: Converting a derived class pointer to a base class pointer. Always safe and implicit. Commonly used in runtime polymorphism.
Base* ptr = new Derived();  // ✅ Upcasting (automatic, safe)

Downcasting (Base → Derived)

Downcasting: Converting a base class pointer back to a derived class pointer. Requires explicit casting. Can be unsafe if object is not actually Derived. Use dynamic_cast for safety.
Base* ptr = new Derived();
Derived* d = dynamic_cast<Derived*>(ptr);  // Safe downcast
if (d != nullptr) {
    d->derivedMethod();   // ✅ Safe to use
}
// If ptr actually points to a Base object:
// dynamic_cast returns nullptr → check before use!

Cross Casting

Cross Casting: Converting a pointer from one sibling class to another sibling class through a common base class. Only possible in multiple inheritance using dynamic_cast.
class Animal { public: virtual void sound() {} };
class Flyable { public: virtual void fly() { cout << "Flying"; } };
class Bird : public Animal, public Flyable {};

Animal* a = new Bird();
Flyable* f = dynamic_cast<Flyable*>(a);  // Cross cast!
if (f) { f->fly(); }  // Output: Flying

DowncastingBase → Derived⚠️ Can faildynamic_cast (returns nullptr if fails)
Cross CastingSibling → Sibling⚠️ Needs checkdynamic_cast via base (multiple inheritance)

Object Serialization in C++
SerializationDeserializationFile Handlingofstream/ifstream

The Problem

Objects exist only in RAM during program execution. When the program ends, all object data is lost permanently. Users must re-enter data every time.

Object Serialization

Object Serialization: The process of converting an object's state (data) into a format that can be stored permanently in a file or transmitted over a network.
Deserialization: The reverse process — reading stored data from a file and reconstructing the original object.

Types of Serialization

TypeStorageAdvantageDisadvantage
Text SerializationHuman-readable textEasy to read and editSlower, larger files
Binary SerializationBinary formatFaster, compactNot human-readable

Complete Example

#include <iostream>
#include <fstream>
#include <string>
using namespace std;

class Student {
    int roll;
    string name;
    float marks;
public:
    void input() {
        cout << "Enter Roll: "; cin >> roll;
        cin.ignore();
        cout << "Enter Name: "; getline(cin, name);
        cout << "Enter Marks: "; cin >> marks;
    }
    void save() {  // Serialize to file
        ofstream fout("student.txt");
        fout << roll << endl << name << endl << marks << endl;
        fout.close();
    }
    void load() {  // Deserialize from file
        ifstream fin("student.txt");
        fin >> roll; fin.ignore();
        getline(fin, name);
        fin >> marks;
        fin.close();
    }
    void display() { cout << roll << " " << name << " " << marks; }
};
int main() {
    Student s1, s2;
    s1.input();
    s1.save();   // Save to file
    s2.load();   // Load from file
    s2.display(); // Same data as s1!
}

Real-World Applications

Banking: Customer account details Online Games: Save progress Shopping Apps: Cart items Student Systems: Student records

One-Line Revision Notes

ofstream = write to file (serialize) ifstream = read from file (deserialize) fstream = both read and write Always call .close() after file operations
